Même si c'est pas vraiment le sujet que je souhaitais aborder (bon d'accord, je n'en avais pas vraiment en tête, mais c'était pas vraiment dans ce domaine), voici un mini-tuto pour pouvoir développer pour les smartphones Android sur Netbeans (qui est, je le rappelle, bien mieux que Eclipse ! #troll).

Première étape, commune et à Netbeans et à Eclipse, il faut télécharger le SDK sur le site officiel de développeurs Android [EN], puis l'installer. Jusqu'ici, rien de bien compliqué... Une fois ceci fait, cherchez l'executable "android" (tools\android.bat sous Windows), puis dans l'onglet Available Packages, sélectionnez le seul site disponible, et installez les platformes / packages que vous désirez installer. Travaillons, par exemple, avec la 2.2. Une chose utile à prendre serait les documentations et quelques exemples pour se mettre dans le bain, à voir si vous débutez ou non.

Une fois installés, allez dans Netbeans, et installez le plugin NBAndroid [EN], disponible sur le site du projet Kenai. Il existe deux façons de l'installer ; ou bien par le package .nbm, ou bien par l'interface des plugins de netbeans, celle que je préfère. Pour cela, allez dans le menu Tools, puis Plugins ; Dans la fenêtre ouverte, sélectionnez l'onglet Settings, puis cliquez sur Add. Entrez l'adresse du dépot de Kenai, à savoir http://kenai.com/downloads/nbandroid/updates.xml. Installez le plugin Android.

Enfin, dernière étape : Toujours dans le menu Tools, allez dans Java Platforms. Ajoutez une nouvelle platforme Java via le bouton correspondant, puis sélectionnez Google Android Open Handheld Platform... Et sélectionnez où vous avez installé le SDK (le dossier extrait). Enfin, dans le prochain écran, sélectionnez la platforme de développement (Android 2.2 en l'occurrence), et puis finir... Et voilà, vous avez terminé.

Vous pouvez désormais créer de nouveaux projets android, les tester sur l'émulateur fourni. Pensez à en créer un auparavant (retournez sur l'outil fourni par l'executable android mentionné plus haut, créez une nouvelle AVD (android virtual device), avec les options que vous voulez (ceci sera peut-être le sujet pour un prochain article ?
